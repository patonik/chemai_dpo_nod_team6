# Multi-Target Prediction Pipeline (IC50, CC50, SI)

Пайплайн для предсказания трёх ключевых параметров противовирусной активности соединений: **IC50** (концентрация полумаксимального ингибирования), **CC50** (цитотоксическая концентрация) и **SI** (индекс селективности).

---

## 📊 Обзор данных и EDA

### Исходные данные
- **Размер выборки**: 751 соединение
- **Количество признаков**: 516 колонок (молекулярные дескрипторы + `index`, `IC50, mM`, `CC50, mM`, `SI`)
- **Тип данных**: числовые (`float64`, `int64`)
- **Пропуски**: отсутствуют (уже предобработано)

### ⚠️ Критические проблемы данных

**1. Признаки с нулевой дисперсией**
- В датасете присутствуют колонки, где все значения одинаковы (например, `NumRadicalElectrons` всегда 0)
- Такие признаки не несут информации, но увеличивают размерность

**2. Отсутствие SMILES-строк**
- Данные содержат **только вычисленные дескрипторы**, но не исходные SMILES
- Это критическое ограничение:
  - Невозможно проверить корректность расчёта дескрипторов
  - Нельзя сгенерировать дополнительные дескрипторы (например, через RDKit)
  - Нельзя выполнить визуализацию химического пространства (t-SNE/UMAP по SMILES)

**3. Экстремальные выбросы**

Анализ выявил соединения с физически сомнительными значениями:

| Проблема | Пример | Потенциальная причина |
|----------|--------|----------------------|
| `IC50` = 0.0004 при `CC50` = 0.7 | Индекс 85 (SI=1750) | Возможна ошибка эксперимента или опечатка |
| `CC50` < 1.0 (единицы) при нормальной активности | Индексы 229, 231, 284 | Нетоксичное соединение не может иметь CC50=0.7 |
| `SI` = 0.06 (менее 1) | Индекс 284 | Биологически невозможно (CC50 < IC50) |
| `IC50` = 3284 (огромное значение) | Индекс 387 | Соединение практически неактивно |
| `CC50` = 0.7 при `IC50` = 0.4 | Индекс 229 | Оба значения подозрительно малы |


**4. Искусственно созданные признаки**
- Дескрипторы типа `BCUT2D_*`, `EState_VSA*` уже являются результатом сложных расчётов
- Генерация новых признаков на их основе (`FeatureGenerator`) может создавать мультиколлинеарность
- **Риск**: Переобучение на шумовых корреляциях

### Целевые переменные
| Переменная | Диапазон | Смысл |
|------------|----------|-------|
| `IC50, mM` | 0.0004 – 3284 | Эффективность (чем меньше, тем лучше) |
| `CC50, mM` | 0.7 – 3280 | Токсичность (чем больше, тем безопаснее) |
| `SI` | 0.06 – 5000 | Индекс селективности = CC50/IC50 |

### Ключевые выводы EDA

**1. Огромный динамический диапазон**
- Значения целевых переменных различаются в **миллионы раз** (от 0.0004 до 3284)
- → Требуется логарифмическая трансформация (`log1p`) для нормализации

**2. Наличие выбросов**
- Экстремально активные соединения: `IC50` ≈ 0.0004, `CC50` > 1000 (отличный профиль)
- Экстремально плохие соединения: `IC50` > 100, `CC50` < 1 (токсичны и неэффективны)
- **Биологически некорректные образцы**: `SI` < 1 (CC50 < IC50) ~10 штук
- → Необходимо клиппирование (отсечение крайних 1% квантилей) и/или удаление аномалий

**3. Структура признаков**
- **Классические дескрипторы**: `MolWt`, `TPSA`, `HeavyAtomCount`, `qed`
- **Топологические индексы**: `Chi`, `Kappa`, `EState`, `BalabanJ`
- **Бинарные фрагменты**: ~450 признаков `fr_*` (наличие функциональных групп)
- **Нулевые колонки**: `NumRadicalElectrons` и возможно другие — требуют удаления

**4. Мультиколлинеарность**
- `MolWt` и `HeavyAtomMolWt` коррелируют почти идеально (r ≈ 0.99)
- `BCUT2D_*` признаки часто сильно коррелированы между собой
- → Требуется отбор признаков или регуляризация

**5. Физические ограничения**
- По определению `SI = CC50 / IC50`, следовательно `CC50 ≥ IC50`
- При `CC50 < IC50` значение SI становится < 1 (биологически неверно)
- → Постобработка должна исправлять такие нарушения

**6. Отсутствие валидации исходных дескрипторов**
- Нет гарантии, что все 516 дескрипторов корректно рассчитаны
- Некоторые дескрипторы могли быть вычислены для невалидных SMILES
- → Желательна кросс-проверка с эталонными значениями

## Описание пайплайна

### Общая схема работы


### 1. Класс `DataCleaner`

**Назначение:** Предобработка и очистка данных.

**Операции:**
- Сохраняет медианы числовых колонок с пропусками для последующего заполнения
- Создаёт бинарные индикаторы пропущенных значений (`{col}_isnull`)
- Конвертирует все колонки в числовой формат через `pd.to_numeric()`
- Заполняет пропуски сохранёнными медианами
- Заменяет `inf` и `-inf` на 0

**Почему это важно:** Многие дескрипторы могут быть не определены для некоторых молекул. Индикаторы пропусков позволяют модели самостоятельно оценить информативность отсутствия значения.

---

### 2. Класс `FeatureGenerator`

**Назначение:** Генерация дополнительных признаков на основе физико-химических дескрипторов молекул.

**Исходные дескрипторы:**
- `MolLogP` — липофильность
- `TPSA` — топологическая площадь полярной поверхности
- `MolWt` — молекулярная масса
- `RingCount` — количество колец
- `BertzCT` — индекс сложности Берца
- `NumRotatableBonds` — количество вращаемых связей
- `NumHAcceptors` / `NumHDonors` — акцепторы/доноры водородных связей
- `HeavyAtomCount` — количество тяжёлых атомов
- `MaxPartialCharge` / `MinPartialCharge` — максимальный/минимальный частичный заряд
- `qed` — количественная оценка drug-likeness
- `FractionCSP3` — доля sp3-гибридизованных атомов углерода

**Генерируемые признаки (24 новых):**
| Признак | Формула | Описание |
|---------|---------|----------|
| `FE_LogP_TPSA` | LogP / TPSA | Отношение липофильности к полярности |
| `FE_LipE` | LogP - TPSA/100 | Липофильная эффективность |
| `FE_Size_Pol` | MolWt × TPSA / 1000 | Размер × полярность |
| `FE_Polar_Density` | TPSA / MolWt × 100 | Плотность полярной поверхности |
| `FE_Flex_LogP` | RotBonds × LogP / 10 | Гибкость × липофильность |
| `FE_Rotatable_Density` | RotBonds / HeavyAtoms | Плотность вращаемых связей |
| `FE_Complexity` | BertzCT / HeavyAtoms | Сложность на тяжёлый атом |
| `FE_Aromatic_Frac` | Rings / HeavyAtoms | Доля ароматических колец |
| `FE_Bertz_Rings` | BertzCT × Rings / 100 | Сложность × кольца |
| `FE_HBond_Total` | HAcc + HDon | Сумма водородных связей |
| `FE_HBond_Density` | (HAcc + HDon) / HeavyAtoms | Плотность H-связей |
| `FE_HBond_Ratio` | HAcc / HDon | Отношение акцепторов к донорам |
| `FE_Charge_Span` | MaxCharge - MinCharge | Размах частичных зарядов |
| `FE_Charge_Magnitude` | \|MaxCharge\| + \|MinCharge\| | Магнитуда зарядов |
| `FE_Charge_Asymmetry` | MaxCharge / \|MinCharge\| | Асимметрия заряда |
| `FE_SP3_Balance` | FracSP3 × (1 - FracSP3) | Баланс sp3-гибридизации |
| `FE_QED_LogP` | QED × LogP | Drug-likeness × липофильность |
| `FE_QED_TPSA` | QED × TPSA | Drug-likeness × полярность |
| `FE_LogP_sq` | sign(LogP) × LogP² | Квадрат LogP с сохранением знака |
| `FE_TPSA_sqrt` | √\|TPSA\| | Корень из TPSA |
| `FE_MW_log` | log(1 + MolWt) | Логарифм молекулярной массы |
| `FE_LogP_MW_ratio` | LogP / log(1+MolWt) | LogP к логарифму массы |
| `FE_HAcc_LogP` | HAcc × LogP | Акцепторы × LogP |
| `FE_HDon_LogP` | HDon × LogP | Доноры × LogP |
| `FE_Rings_LogP` | Rings × LogP | Кольца × LogP |

**Обработка выбросов:** На этапе `fit()` вычисляются границы клиппирования (по умолчанию 1-й и 99-й перцентили) для каждого нового признака. На этапе `transform()` значения обрезаются по этим границам.

### 3. Функция `train_target_model()`

**Назначение:** Полный цикл обучения ансамбля для одной целевой переменной.

#### Этапы работы:

**3.1. Предобработка целевой переменной**
- Для IC50 и CC50 применяется логарифмирование: `y = log(1 + y_raw)`
- SI не логарифмируется (хотя в скрипте используется `log1p` и для него)
- Удаление других целевых переменных из признакового пространства

**3.2. Кросс-валидация (5 фолдов)**
Для каждого фолда выполняется:
1. Клиппирование целевой переменной по 1-му и 99-му перцентилям
2. Yeo-Johnson Power Transformation для нормализации распределения
3. Очистка признаков через `DataCleaner`
4. Генерация дополнительных признаков через `FeatureGenerator`
5. Масштабирование через `QuantileTransformer` (normal distribution)
6. Отбор лучших `k=min(200, n_features)` признаков через `SelectKBest` с `mutual_info_regression`
7. Обучение 4 моделей ансамбля

**3.3. Модели ансамбля**

| Модель | Ключевые параметры |
|--------|-------------------|
| **XGBoost** | 400 деревьев, max_depth=5, lr=0.03, subsample=0.85, colsample=0.85, alpha=0.1, lambda=1.0, early_stopping=50 |
| **LightGBM** | 400 деревьев, max_depth=5, lr=0.04, num_leaves=31, subsample=0.85, colsample=0.85, alpha=0.05 |
| **ExtraTrees** | 350 деревьев, max_depth=10, min_samples_split=10, min_samples_leaf=5 |
| **GradientBoosting** | 350 деревьев, max_depth=5, lr=0.03, subsample=0.85, n_iter_no_change=10 |

**3.4. Оптимизация весов ансамбля**
- Минимизация RMSE на out-of-fold предсказаниях с L2-регуляризацией
- Метод оптимизации: SLSQP
- Ограничения: сумма весов = 1, каждый вес ∈ [0.05, 0.95]
- Если один вес > 0.5, возврат к равным весам (защита от переобучения)

**3.5. Финальное обучение**
- Все трансформеры переобучаются на полном датасете
- Модели обучаются на всех данных
- Тестовые предсказания — взвешенное среднее ансамбля

---

### 4. Функция `post_process()`

**Назначение:** Обеспечение физической согласованности предсказаний.

**Правила:**
1. **Клиппирование в разумные диапазоны:**
   - IC50: [0.001, 5000]
   - CC50: [0.5, 10000]
   - SI: [0.01, 10000]

2. **Исправление нарушения CC50 ≥ IC50** (цитотоксичность не может быть ниже противовирусной активности):
   - Сильное нарушение (IC50 > 2×CC50): CC50 = IC50 × 1.5
   - Умеренное нарушение: CC50 = max(CC50, IC50 × 0.9)

3. **Корректировка SI** на основе фактического отношения CC50/IC50:
   - Если расхождение > 100%: SI = 0.3 × SI_pred + 0.7 × (CC50/IC50)

### 5. Выходной формат

Файл `submission.csv` содержит три колонки:
- `index` — идентификатор образца
- `IC50` — предсказанная концентрация полумаксимального ингибирования
- `CC50` — предсказанная цитотоксическая концентрация
- `SI` — предсказанный индекс селективности
